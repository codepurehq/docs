# Supported Languages & Frameworks

Codepure’s Static Application Security Testing (SAST) engine is built to find and prioritize security issues directly in your source code. Designed for modern, developer-centric workflows, it delivers blazing-fast results with minimal noise.

---

## How Codepure SAST Works

Codepure analyzes your source code to identify and prioritize real security vulnerabilities before they reach production. Our engine is specifically tuned to integrate seamlessly into modern developer workflows, focusing on accuracy and performance.

* **Developer-First Speed:** Engineered for rapid execution, Codepure scans your codebase in seconds, providing instant, actionable feedback directly within your CI/CD pipelines.
* **Precision Scanning:** The engine intelligently traces data pathways across your application, accurately identifying areas where untrusted input could lead to critical vulnerabilities like injections or unauthorized data access.
* **Low False Positives:** Codepure understands the context of your code. It recognizes when you have already applied safe coding practices—such as data validation, type casting, or encoding—and automatically filters out those safe patterns so you only spend time on genuine threats.
* **Comprehensive Secret Detection:** Beyond structural vulnerabilities, Codepure actively hunts for high-risk hardcoded credentials, API keys, and tokens across your repositories, protecting your infrastructure from accidental exposure.

## Language Support Matrix

Codepure natively parses the Abstract Syntax Tree (AST) of the following languages. Below is a detailed breakdown of the file types, frameworks, and technologies officially supported by our security engine.
