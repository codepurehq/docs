## How Codepure SAST Works

Codepure utilizes **Intra-procedural Taint Analysis** combined with robust, language-specific secret scanners. 



* **Developer-First Speed:** By tracking the flow of untrusted data strictly within the boundaries of a single file or function, the engine scans millions of lines of code in seconds, making it ideal for real-time CI/CD pipelines.
* **Taint Tracking:** The engine maps data flows from vulnerable **sources** (like HTTP requests or UI inputs) through to dangerous **sinks** (like database queries or OS commands).
* **Smart Sanitization:** To drastically reduce false positives, Codepure intentionally drops taint flags if data passes through safe cleaning functions (such as numeric casts or HTML encoders).
* **Advanced Secret Scanning:** For mobile and modern web apps, the engine utilizes entropy-based and pattern-based stateless scanning to detect high-risk, hardcoded credentials.
